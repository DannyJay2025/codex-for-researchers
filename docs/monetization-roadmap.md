# Monetization Roadmap

Codex for Researchers should stay useful as an open-source project while giving serious users a clear paid path.

## Product Ladder

| Tier | Offer | Buyer | Delivery |
| --- | --- | --- | --- |
| Free | Core Codex skills, basic templates, sample prompts | Individual researchers | Public GitHub repo |
| Premium | Premium templates, field packs, private examples | Power users, students, consultants | WeChat payment plus manual delivery |
| Service | Done-with-you audits and manuscript support | Authors, labs, startups | Fixed-scope service packages |
| Lab | Private Codex plugin customized to lab SOPs | PIs, research groups, R&D teams | Private repo or organization plugin |

## Free To Paid Conversion

Use the public repo to prove quality:

- Keep the core skills genuinely useful.
- Show example outputs without exposing the deepest paid templates.
- Add clear calls to action in the README and docs.
- Use issues and discussions to collect workflow requests.
- Convert repeated requests into paid templates or field packs.
- Use WeChat payment for early buyers before building a full checkout system.

## Paid Assets To Build First

1. Journal submission kit: abstract polish, cover letter, response letter, checklist.
2. Citation audit kit: claim splitter, citation verifier, high-risk claim report.
3. Reproducibility kit: repository audit, availability statements, release checklist.
4. Field packs: biomedical, AI/ML, and social science workflows.
5. Lab onboarding kit: private writing style guide, data policy, and manuscript SOP.

## 30-Day Launch Plan

Week 1:

- Publish the public repo.
- Add 3 example outputs from non-sensitive sample text.
- Add a WeChat QR code or manual contact path for payment.

Week 2:

- Build the first premium template pack in a private repo.
- Offer a founding-buyer price for the first 10 customers.
- Ask early users for one painful workflow each.

Week 3:

- Package the first paid service: citation audit or reproducibility audit.
- Add a public issue template for paid workflow requests.
- Publish one short demo video or GIF.

Week 4:

- Create the first lab customization offer.
- Convert common support work into docs.
- Raise pricing for new buyers once the workflow is proven.
