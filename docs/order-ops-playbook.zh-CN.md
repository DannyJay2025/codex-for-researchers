# 订单运营手册

这份手册用于把低价引流单变成可重复交付的个人服务流程。核心原则：先确认付款和范围，再创建本地订单目录；客户文件只放在本地忽略目录，不提交到公开仓库。

## 订单状态

每一单只允许处于一个状态：

| 状态 | 含义 | 下一步 |
| --- | --- | --- |
| `lead` | 对方咨询但未付款 | 发服务菜单和边界说明 |
| `paid_waiting_materials` | 已付款但材料不全 | 催客户补材料 |
| `ready` | 付款、材料、范围都确认 | 创建订单工作区并开始交付 |
| `working` | 正在处理 | 使用对应 skill 和交付模板 |
| `delivered` | 已交付初版 | 等待一次澄清或确认 |
| `closed` | 已确认完成 | 记录是否可脱敏做案例 |
| `refunded` | 已退款或作废 | 记录原因，不再处理 |

## 收单流程

1. 客户加微信 `zhiyanaishe`。
2. 发送服务菜单、价格和边界。
3. 客户选择服务并付款。
4. 客户发送付款截图和材料。
5. 判断材料是否在低价单范围内。
6. 创建本地订单工作区。
7. 完成交付并保存最终版。
8. 请求 star、转介绍或后续小包。

## 本地订单工作区

创建订单：

```powershell
python scripts\create_order_workspace.py --service paper-reading --buyer buyer-name
```

可选服务：

```text
paper-reading
citation-audit
repro-audit
```

脚本会在被 git 忽略的 `orders/` 下创建：

```text
orders/<order-id>/
  input/
  output/
    draft.md
  status.json
  delivery-checklist.md
  notes.md
```

使用规则：

- `input/` 放客户材料。
- `output/draft.md` 是工作草稿。
- `output/final.md` 是最终交付。
- `status.json` 记录付款、材料、范围和交付状态。
- `delivery-checklist.md` 是交付前质检表。

## 订单追踪

可以从 `ops/order-tracker-template.csv` 复制一个本地私有追踪表，保存为：

```text
orders/order-tracker.csv
```

每次客户咨询或付款后更新一行。不要把真实客户姓名、微信号、付款截图、论文或稿件内容提交到公开仓库。

建议字段：

- `order_id`
- `date`
- `buyer_alias`
- `service`
- `price_rmb`
- `status`
- `due_date`
- `materials_status`
- `delivery_path`
- `public_sample_consent`
- `notes`

## 范围判断

低价引流单只接最小范围：

| 服务 | 接受范围 | 超出范围 |
| --- | --- | --- |
| 论文精读 | 一篇论文的链接、摘要、引言或核心片段 | 多篇系统综述、完整翻译、深度统计复核 |
| 引用审计 | 一段 300-800 字文本或少量 claims | 完整 manuscript、系统综述、真实文献检索承诺 |
| 复现包审计 | 一个 repo 的 README、目录结构或简要说明 | 完整跑通实验、修复代码、重做全部结果 |

超出范围时不要硬接低价单。回复：

```text
这部分超出当前低价单范围。我们可以先做一个最小切口：一篇论文、一段文字，或一个 repo 的 README/目录结构。完整稿件或完整项目审计需要单独报价。
```

## 退款和作废

建议保持简单：

- 未开始处理且客户材料明显不适合：可退款或请客户换材料。
- 已开始处理但客户临时大幅改范围：当前单继续按原范围交付，新范围另下单。
- 客户要求代写、伪造引用、保证发表：拒绝并退款。
- 客户要求处理敏感隐私数据：先要求脱敏，无法脱敏则拒绝。

## 交付质检

每次交付前确认：

- 输入范围写清楚。
- 不承诺发表、合规或引用绝对正确。
- 未核实事实已标注。
- 不编造文献。
- 不把客户材料放入公开案例。
- 给出一个明确下一步。

## 复购路径

首单交付后，只在客户认可结果时推荐：

```text
如果你后面还有连续需求，可以做小包：

19.9 元：10 篇论文快速精读
39.9 元：一节 manuscript 引用审计
99 元：一个完整复现包审计

先不用急着买，等你确认这次结果有用再说。
```

## 每日 20 分钟运营

1. 发一条推广文案。
2. 回复微信咨询。
3. 更新本地订单追踪表。
4. 处理一单最小交付。
5. 把重复问题补进 README、FAQ、模板或营销文案。
